Генератор конфигурации роутеров или свитчей на основании шаблонов.

Вызов config.bat [-h, -ip, -t иvя_шаблона, -c имя_конфигурации.yaml]
-h - вызов короткой справки, ее вывод дублируется также при вызове команды без параметров

-t выбрать шаблон для настройки, по-умолчанию sw_config.txt
-c выбрать файл с конфигурацией свитча, по-умолчанию sw_data.yaml
-ip имя файла берем из hostname
-h вывод этой справки

В файле конфигурации обратить внимание на синтаксис
Каждый новый девайс должен начинатся со знака "-" далее пробел
Все последующие строчки должны начинаться с двух пробелов

Формат файла
    Имя_параметра: параметр можно с пробелами
    в имени параметра не должно быть пробелов или специальных символов,
    допускается символ _

Работа с шаблонами.
    1. Копируем шаблон конфигурации из готового устройства и сохраняем для удобства в файл txt.
    2. Значения, которые необходимо будет менять заносим в файл с конфигурацией
     в формате ПРОИЗВОЛЬНОЕ_ИМЯ: ЗНАЧЕНИЕ (регистр букв имеет значение)
    3. В шаблоне данные, которые необходимо менять, берем в {{двойные_фигурные_скобки}},
     в которых пишем ПРОИЗВОЛЬНОЕ_ИМЯ без пробелов и кавычек в том регистре, в каком оно
     указано в файле данных.
    4. При запуске скрипта в эти места будут проставлены ЗНАЧЕНИЯ из файла с данными.
     Новый файл конфигурации будет сохранен с именем hostname.
     Если выбран параметр -ip имя будет в виде значения параметра ip_vlan_intf до пробела (IP-адрес)).

Сисок параметров в файле yaml
      1. last_interface номер последнего порта FastEthernet0/n
      2. sub_interface_vlan_name используется для генерации дополнительных интерфейсов vlan
        где vlan номер влана и name имя влана
      3. uplink_gig порт аплинка GiabitEthernet
      4. no_uplink порты GiabitEthernet, которые не являются аплинками
      5. uplink_vlan список вланов на порту uplink_gig
      6. voice_vlan для настройки портов FastEthernet
      7. access_list номер аксесс-листа, который будет настраиваться
      8. access_list_rule параметры аксесс-листа, permit и/или remark или deny
      9. ip_vlan_intf ip и маска на интерфейсе source_interface
     10. source_interface интерфейс, который исользуется вместо Vlan1

Можно использовать шаблон попроще как в файле config1.txt и к нему файл config.yaml. Параметры можно создавать самому и вписывать в нужные места в
{{двойые фигурные скобки}}. Или можно использовать более "упоротый" файл шаблона с циклами и условиями sw_config.txt и sw_data.yaml

В каталоге venv находится виртуальное окружение Python с установленными пакетами, не удалять.

Зависимости:
    Python 3.10.2
    jinja2
    pyyaml
